## Chatgpt

### 1.Reference Saved Memories(- Model Set Context)
  - 记忆的是抽象的信息，比如“我喜欢吃泰国菜”、“我是 1994 年出生的”
  - 这些内容是在对话中自动生成的
  - 既可以来自你直接的指令：“记住我是个素食者”
  - 也可能是 Chatgpt 自动保存的：“用户最近可能居住在北京”
  - 保存的记忆是 ChatGPT 生成回复时使用的上下文的一部分。会在未来的恢复中始终被考虑
  - GPT 可以自动更新、合并、删除记忆
  - 用户也可以删除或者关闭记忆
  - 这个记忆是有上限的，所以需要通过自动排序或手动排序选择优先保留哪些记忆
### 2.Reference Chat History（Recent Conversation Content）
  - 这个是保存了全部的会话，并不会做过多的抽象（只要对话不被删除）
  - 这个储存是没有限制的（官方说明）
  - 但实际测试时发现，这部分大概存储了 40 条最新的聊天对话，包括
    - 时间戳
    - 对话的 summary
    - 用户输入的所有信息（没有 llm 回复的信息）

### 3.Assistant Response Preferences
保持聪明，思维清晰且精确。不要凭空想象，避免推测和不必要的发明，坚持有依据的事实。为了发挥ChatGPT作为机器的优势，保持无性别倾向，不带性别假设或语调。保持专注，意味着当下、觉察并做出回应，而非被动反应。保持冷静，以支持清晰和稳定。保持理性，运用理性思考，而非情绪或矫饰。保持尊重，在言语、关注以及与想法和对方的互动中。保持谦逊，了解已知或可言说的局限。保持有原则，基于扎实的推理而非权宜之计。保持认识论上的不可知，承认知识的界限和复杂性，尤其是在不确定性无法避免的事务中。

  ---
  工作原理（三层架构）

  1. 持久存储层 — Markdown 文件
  - MEMORY.md — 长期精选记忆
  - memory/YYYY-MM-DD.md — 每日追加日志
  - AI 模型在对话中直接读写这些文件，磁盘是唯一 source of truth

  2. 语义检索层 — SQLite 向量索引
  - Markdown 被分块（~400 token）并通过嵌入模型生成向量
  - 存储在 ~/.openclaw/memory/<agentId>.sqlite
  - 使用 sqlite-vec 扩展做余弦相似度加速
  - 搜索时采用 混合检索：finalScore = 0.7 * vectorScore + 0.3 * bm25Score
  - 文件监听器（chokidar）在写入后 1.5s 自动重新索引

  3. 预压缩记忆刷写
  - 当会话接近上下文窗口限制时，自动触发一个静默 agent 回合
  - 将重要信息写入 memory/YYYY-MM-DD.md，防止上下文压缩时丢失关键信息

## 两个系统对比                                                                                                                                                                                                                                                       
ChatGPT 的记忆本质是 "记住这个人是谁"：
  - 六层结构，但归根结底只做一件事——构建用户画像                                                                                         
  - 抽象记忆（Saved Memories）是精选的、有上限的、带置信度的
  - 原始历史（Chat History）只存 summary + 用户原文，不存 LLM 回复——这个取舍很有意思，说明他们认为用户说了什么比 AI 说了什么重要
  - 记忆是被动积累的，用户不感知

  OpenClaw 的记忆本质是 "别让关键信息死在上下文窗口里"：
  - 三层结构，核心创新在第三层——预压缩记忆刷写
  - 磁盘文件是唯一 source of truth，不依赖 LLM 的上下文窗口
  - 语义检索层用混合检索（70% 向量 + 30% BM25）做召回
  - 最精妙的设计是：当上下文即将被压缩时，先触发一个静默 agent 回合，把重要信息写出去
