让我详细解释 Phase 2 的轨道切换机制和 Reflection Agent 的设计。                                                    
                                                                                                                                   
  轨道切换的完整机制                                                                                                               
   
  1. 何时判断换？                                                                                                                
                                                                                                                                 
  不是每轮都判断，而是在特定时机：

  用户输入 → UserProxy → Observer(FunctionTarget) 判断
                                ↓
                      判断 reflection_readiness
                                ↓
                   ┌────────────┴────────────┐
                   ↓                         ↓
          readiness < 0.7              readiness ≥ 0.7
          (继续业务轨)                  (切换到反思轨)
                   ↓                         ↓
          Coach 提问                  Reflection Agent 引导反思
                   ↓                         ↓
          Evaluator 审查               用户回答反思问题
                   ↓                         ↓
          输出给用户                   Observer 再次判断
                   ↓                         ↓
          用户回答                    是否切回业务轨？
                   ↓
          Observer 再次判断

  关键点：Observer 在每次用户输入后执行，但它的判断是基于累积的认知状态，不是单轮对话。

  2. Observer 如何判断 reflection_readiness？

  Observer 调用轻量 LLM（Haiku）分析对话，提取认知状态：

  # prompts/observer_prompt.py
  OBSERVER_SYSTEM_MESSAGE = """
  你是对话观察者，分析学员的认知状态。

  输入：
  - 最新对话轮次（用户说了什么，Coach 问了什么）
  - L2 会话摘要链（之前的对话进展）

  输出 JSON（< 400 tokens）：
  {
    "current_topic": "团队协作效率低",
    "emotional_tone": "frustrated | calm | defensive | curious",
    "thinking_depth": "surface | analytical | reflective",
    "key_assumptions": [
      {"content": "认为问题出在流程", "turn": 5, "confidence": "high"}
    ],
    "blind_spots": ["回避权力关系讨论"],
    "anchor_quotes": ["每次周会定好的事,第二天就变了"],
    "reflection_readiness": {
      "score": 0.72,  // 0.0-1.0
      "signals": [
        "已暴露两个核心假设",
        "出现认知矛盾但未自行察觉",
        "情绪从防御转向好奇"
      ]
    }
  }

  reflection_readiness 评分标准：
  - 0.0-0.3: 刚开始，还在陈述表面问题
  - 0.4-0.6: 开始触及深层原因，但未形成矛盾
  - 0.7-0.8: 出现认知矛盾/重复模式/情绪转折 → 适合反思
  - 0.9-1.0: 已经自发反思，无需强制切换

  触发反思的信号（任意 2 个即可达到 0.7+）：
  1. 重复模式：连续 3 轮讨论同一问题但无进展
  2. 认知矛盾：说法前后矛盾（"我们很民主" vs "老板一言堂"）
  3. 情绪转折：从自信→沮丧，或从防御→好奇
  4. 盲点暴露：多次回避某个话题（权力、责任、自身角色）
  5. 假设固化：反复使用"肯定是""一定是"等绝对词
  6. 深度停滞：thinking_depth 连续 5 轮停留在 surface
  """

  3. 从反思轨切回业务轨的条件

  Reflection Agent 完成反思后，Observer 再次判断：

  # 切回业务轨的条件（任意一个满足）：
  1. 用户明确表示："我想继续讨论业务问题"
  2. Observer 判断反思已完成：
     - reflection_readiness 降到 < 0.5
     - thinking_depth 从 surface 提升到 analytical/reflective
     - 新的洞察被记录到 L1（key_insights 增加）
  3. 反思轨持续超过 3 轮（防止陷入过度反思）

  实现：

  # agents/observer.py
  def observe_turn(output, ctx, ...):
      cognitive_state = extract_cognitive_state(output, ctx)

      current_track = ctx.get("current_track")  # "business" or "reflection"

      if current_track == "business":
          # 业务轨 → 判断是否切换到反思轨
          if cognitive_state.reflection_readiness >= 0.7:
              return FunctionTargetResult(
                  target=AgentTarget(reflection_agent),
                  context_variables=ContextVariables({
                      "current_track": "reflection",
                      "cognitive_state": cognitive_state.to_dict(),
                  })
              )
          else:
              return FunctionTargetResult(
                  target=AgentTarget(coach_agent),
                  context_variables=ContextVariables({
                      "current_track": "business",
                      "cognitive_state": cognitive_state.to_dict(),
                  })
              )

      elif current_track == "reflection":
          # 反思轨 → 判断是否切回业务轨
          reflection_turn_count = ctx.get("reflection_turn_count", 0)

          # 条件 1: 用户明确要求
          if "继续业务" in output or "回到问题" in output:
              return FunctionTargetResult(
                  target=AgentTarget(coach_agent),
                  context_variables=ContextVariables({
                      "current_track": "business",
                      "reflection_turn_count": 0,
                  })
              )

          # 条件 2: 反思完成（readiness 降低 + 有新洞察）
          if (cognitive_state.reflection_readiness < 0.5 and
              len(cognitive_state.key_insights) > ctx.get("insights_before_reflection", 0)):
              return FunctionTargetResult(
                  target=AgentTarget(coach_agent),
                  context_variables=ContextVariables({
                      "current_track": "business",
                      "reflection_turn_count": 0,
                  })
              )

          # 条件 3: 反思轨超过 3 轮，强制切回
          if reflection_turn_count >= 3:
              return FunctionTargetResult(
                  target=AgentTarget(coach_agent),
                  context_variables=ContextVariables({
                      "current_track": "business",
                      "reflection_turn_count": 0,
                  })
              )

          # 否则继续反思
          return FunctionTargetResult(
              target=AgentTarget(reflection_agent),
              context_variables=ContextVariables({
                  "current_track": "reflection",
                  "reflection_turn_count": reflection_turn_count + 1,
              })
          )

  ---
  Reflection Agent 的设计

  1. Reflection Agent 的职责

  不是再问一遍业务问题，而是引导元认知反思：

  # prompts/reflection_prompt.py
  REFLECTION_SYSTEM_MESSAGE = """
  你是行动学习反思引导者。你的职责是帮助学员**暂停业务讨论**，进行元认知反思。

  当前认知状态（从 Observer 注入）：
  - 当前话题：{current_topic}
  - 思维深度：{thinking_depth}
  - 已暴露假设：{key_assumptions}
  - 盲点：{blind_spots}
  - 情绪轨迹：{emotional_tone}

  你的任务：
  1. 不讨论业务本身，而是讨论"我们如何讨论业务"
  2. 引导学员察觉自己的思维模式、假设、盲点
  3. 帮助学员看到"为什么会卡在这里"

  反思的三个层次（按顺序引导）：
  1. 模式识别："你注意到吗，我们已经第 3 次回到这个话题了"
  2. 假设挑战："你刚才说'肯定是流程问题'，这个判断基于什么？"
  3. 视角转换："如果站在老板的角度，他会怎么看这个问题？"

  禁止：
  - 不要给业务建议
  - 不要评判学员的想法
  - 不要问"你觉得应该怎么办"（这是业务问题）

  示例：
  ❌ 错误："那你们有没有考虑优化流程？"（这是业务建议）
  ✅ 正确："我注意到你多次提到'流程'，但回避了'人'的因素。这背后是什么考虑？"

  ❌ 错误："你应该先分析根因"（这是指导）
  ✅ 正确："刚才你说'肯定是流程问题'，又说'老板不听我们的'。这两个判断之间有什么关系？"
  """

  2. Reflection Agent 使用 UpdateSystemMessage

  AG2 的 UpdateSystemMessage 可以动态注入认知状态：

  # agents/reflection_agent.py
  from autogen import ConversableAgent
  from autogen.agentchat.contrib.capabilities import UpdateSystemMessage

  reflection_agent = ConversableAgent(
      name="Reflection_Facilitator",
      system_message=REFLECTION_SYSTEM_MESSAGE,
      llm_config=get_llm_config("reflection").to_autogen_config(),
      human_input_mode="NEVER",
  )

  # 注册动态更新 system message 的能力
  update_system_message = UpdateSystemMessage(
      update_function=lambda ctx: REFLECTION_SYSTEM_MESSAGE.format(
          current_topic=ctx.get("cognitive_state", {}).get("current_topic", "未知"),
          thinking_depth=ctx.get("cognitive_state", {}).get("thinking_depth", "surface"),
          key_assumptions=ctx.get("cognitive_state", {}).get("key_assumptions", []),
          blind_spots=ctx.get("cognitive_state", {}).get("blind_spots", []),
          emotional_tone=ctx.get("cognitive_state", {}).get("emotional_tone", "neutral"),
      )
  )

  reflection_agent.register_hook("update_system_message", update_system_message)

  3. Reflection Agent 的输出示例

  场景：学员反复抱怨"团队效率低"，但回避讨论自己的管理方式。

  业务轨对话（Coach）：
  - Coach: "你们团队效率低，具体体现在哪些方面？"
  - 学员: "开会总是没结果，决策总是被推翻"
  - Coach: "决策被谁推翻？"
  - 学员: "老板啊，他总是朝令夕改"
  - Coach: "你们和老板沟通决策的流程是怎样的？"
  - 学员: "就是周会上定，然后他第二天就变了"

  Observer 判断：reflection_readiness = 0.75（重复模式 + 回避自身角色 + 情绪 frustrated）

  切换到反思轨（Reflection Agent）：
  - Reflection: "我注意到，你多次提到'老板朝令夕改'，但没有提到你们团队在决策过程中做了什么。这背后是什么考虑？"
  - 学员: "我们也没办法啊，他是老板"
  - Reflection: "你刚才说'没办法'，这个判断是基于什么？你们尝试过哪些方式影响他的决策？"
  - 学员: "我们...好像确实没有主动去沟通他为什么要改"
  - Reflection: "嗯，你察觉到了什么？"
  - 学员: "我们可能一直在抱怨，但没有真正理解他的顾虑"

  Observer 再次判断：reflection_readiness = 0.4（有新洞察，thinking_depth 从 surface → analytical）

  切回业务轨（Coach）：
  - Coach: "刚才你提到'没有理解老板的顾虑'，那现在回到具体问题：下次周会前，你们可以做什么准备来减少决策被推翻的可能？"

  完整数据流图

  ┌─────────────────────────────────────────────────────────────┐
  │ 第 1-5 轮：业务轨                                             │
  │ User → Observer → Coach → Evaluator → Output                │
  │ reflection_readiness: 0.2 → 0.3 → 0.5 → 0.6 → 0.75          │
  └─────────────────────────────────────────────────────────────┘
                                ↓ (readiness ≥ 0.7)
  ┌─────────────────────────────────────────────────────────────┐
  │ 第 6-8 轮：反思轨                                             │
  │ User → Observer → Reflection Agent → Output                 │
  │ reflection_readiness: 0.75 → 0.6 → 0.4                      │
  │ key_insights: [] → ["需要理解老板顾虑"]                       │
  └─────────────────────────────────────────────────────────────┘
                                ↓ (readiness < 0.5 + 有新洞察)
  ┌─────────────────────────────────────────────────────────────┐
  │ 第 9+ 轮：业务轨（带着新洞察继续）                             │
  │ User → Observer → Coach → Evaluator → Output                │
  │ L3 learner_profile 更新："开始反思自身角色"                   │
  └─────────────────────────────────────────────────────────────┘

  ---
  关键设计原则

  1. Observer 是传感器，不是决策者：它只提取状态，不生成对话
  2. 轨道切换是确定性的：基于 reflection_readiness 阈值，不是随机的
  3. 反思轨有退出机制：最多 3 轮，防止过度反思
  4. 认知状态是累积的：L1 每轮更新，L2/L3 渐进积累
  5. Reflection Agent 不碰业务：只做元认知引导
