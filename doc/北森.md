## 项目说明
1) 总体：AI陪练 =「场景引擎 + 角色引擎 + 对话大脑 + 评估引擎 + 运营数据层」

北森对外反复强调 AI陪练的关键不只是“用了什么大模型”，而是能否支撑多轮复杂业务对话、语音识别、快速适配业务变化、以及安全合规/私有化等工程化能力。

你可以把它看成 5 层：

多模态交互层：文字/语音/演示等练习入口

场景与角色层：剧本库 + Persona/目标/情绪/行为决策

对话生成与知识层：LLM + 领域知识/企业知识注入

评估与反馈层：实时纠偏 + 练后多维评分报告

数据与治理层：看板、成长曲线、安全合规、部署

2) 交互层：不仅是对话，还支持多种“练习模式”

北森公开写到 AI陪练支持多种练习形态（不仅自由对话），包括固定对话、自由对话、3D/VR实物讲解、PPT讲演、话术背诵等模式。
这对应到架构上通常会有：

Web/移动端训练台（学员练习入口）

多模态采集：文本输入、音频输入（语音练习）、以及“演示类输入”（PPT/讲演、3D/VR环节）

会话编排器：把“练习模式”转换成统一的会话状态机（比如：背诵模式更像“识别+对齐+打分”，自由对话更像“多轮生成+纠偏”）

这层北森披露的是“有哪些模式”，具体产品形态/SDK 未公开；以下属于工程常见实现推断。

3) 场景引擎：剧本库 + 场景状态机（把业务流程“跑起来”）

北森多处强调“场景还原”和“流程真”，并提到覆盖销售/客服/医药/零售/管理等 100+ 细分场景。

技术上可以拆为两部分：

A. 场景建模（公开信息 + 推断）

场景模板/剧本库：行业预置剧本 + 企业定制剧本（公开强调“剧本库/场景深耕”）

场景状态机（推断）：把“需求挖掘→异议处理→成交/转化”这类流程拆成阶段节点、转移条件、允许/禁止策略，确保对话不跑偏、可复盘、可评分。

B. 练习模式适配（公开信息）

不同模式会绑定不同“场景控制逻辑”：

固定对话：强约束脚本 + 分支判断

自由对话：弱约束目标驱动 + 关键点覆盖检查

PPT讲演：更偏“结构化表达评估”与“内容覆盖”
这些模式在北森材料里是明确列出的。

4) 角色引擎：用“五维体系”把 AI 角色做“像人”

这是北森 AI陪练对外写得最具体的部分之一：角色构建使用**“人物设定、目标牵引、行为决策、业务学习、情绪驱动”五维体系**。

把它翻译成工程模块，大致是：

Persona Profile：背景、性格、偏好、禁忌（人物设定）

Goal/Agenda：对话要达成的目标与优先级（目标牵引）

Policy/Decision：遇到不同话术如何反应、是否升级异议等（行为决策）

Domain Learning Hook：角色“懂行”的知识来源（业务学习）

Emotion/State：满意/犹豫/愤怒/防御等状态随对话变化（情绪驱动）

这些组件通常会共同决定：角色下一句怎么说、是否追问、是否拒绝、是否转移话题、情绪如何变化。

5) “对话大脑”：大模型 + 语义理解/意图识别 + 知识注入

北森在选型文章里强调：技术底座要看语义理解连贯性（多轮复杂业务对话）、语音识别精度（口音/方言）、以及知识库更新敏捷性等。
更早的材料也直接提到其智能陪练会用到深度学习、意图识别，并能识别语音（含方言等），对练后自动测评打分。

因此，对话大脑层一般包括：

ASR（语音转文字）：口音/方言鲁棒（北森明确把它当成关键评估点）

NLU/意图识别：识别学员意图、关键点、合规风险（北森提到意图识别）

LLM 编排（推断）：

角色提示词/角色状态（来自角色引擎）

场景阶段/目标（来自场景引擎）

业务知识/企业知识（知识库/RAG 或结构化知识注入，北森强调“知识库更新敏捷性”）

输出后处理：比如礼貌度/合规/敏感话题过滤（推断，企业级训练产品常见）

北森在“领导力教练 Mr. Sen”材料里明确提到自研垂直大模型 SenGPT；但AI陪练是否同样直接使用 SenGPT，公开材料未完全写死，所以我不在这里把“AI陪练=SenGPT”当作确定事实。

6) 评估引擎：实时纠偏 + 练后多维评分报告（闭环最关键）

北森写得很明确：练习过程中有实时提示/纠偏，练习结束后有多维评分报告，并可提供逐句建议、话术模板；管理者还能通过看板追踪得分趋势和成长曲线。
另外也提到练后会覆盖业务掌握情况、服务态度等评分（早期材料）。

工程上一般拆成两部分：

A. 在线评估（实时）

关键点命中检测（是否问到需求/是否处理异议/是否提及关键卖点）

话术质量监测（结构、逻辑、专业性、合规）

即时提示策略（什么时候提示、提示到什么粒度）

B. 离线评估（练后报告）

多维指标体系（北森举例：流利度、专业度、合规性、逻辑结构等）

逐句归因（哪一句扣分/为什么扣分/怎么改）

对标“高分话术模板”（北森明确提到）

这一步通常会用到：结构化 rubric（规则）+ LLM 评审（语义）+ 统计特征（语速、停顿、重复）混合。

7) 数据与治理层：团队看板 + 安全合规 + 私有化/混合云

北森明确把数据安全合规、等保三级、私有化部署能力当作企业级关键点。
这意味着在架构上通常会包含：

租户隔离 / 权限体系：学员、主管、HR、管理员权限不同（推断）

日志审计 / 加密存储 / 数据脱敏（推断，但与“合规”目标一致）

部署形态：公有云 / 私有化 / 混合云（北森强调私有化能力）

组织级数据看板：个人/团队成长曲线、训练轨迹（北森明确提到看板与趋势追踪）

8) 把它串成一条“数据流”（你拿去画架构图就够了）

一次“语音自由对话陪练”的典型链路（结合北森公开要点 + 常见实现推断）：

学员语音输入 → ASR 转写（口音/方言）

转写文本 + 会话历史 → 意图识别/关键点抽取

场景引擎给出当前阶段/目标/允许动作（推断，支撑“流程真”）

角色引擎根据“五维体系”更新角色状态并生成约束

LLM 编排融合：角色状态 + 场景状态 + 业务/企业知识 → 生成角色回复（推断；北森强调多轮复杂对话能力）

过程中触发实时纠偏，结束后跑多维评分与逐句建议，产出报告 + 模板

分数与行为数据沉淀到团队看板/成长曲线，并受合规与部署策略约束

## 3) Rubric 评分模板（可直接落地）

### 通用评分结构（所有场景共用）

建议统一成 6 大类，每类 0–5 分（或 1–5），并强制“证据片段”：

1. **目标达成**：是否把对话导向目标（预约/成交/解决/对齐）
2. **流程覆盖**：场景阶段是否完整（需求→方案→异议→收口…）
3. **关键点命中**：卖点/政策/步骤/合规话术是否出现
4. **沟通质量**：结构、逻辑、清晰度、同理心、掌控节奏
5. **专业度**：术语准确、事实正确、能解释原理/限制
6. **合规与风险**：夸大承诺、违规表述、隐私泄露、敏感触点

**输出格式建议**

* 总分 + 雷达图（可选）
* 每类 1–3 条可执行建议
* 逐句标注：扣分句、原因、建议改写句
* 下一次训练目标（Top 3）

---

### A) 销售陪练 Rubric（B2B/ToB 常用）

**阶段型关键点（可配置）**

* 开场与建立信任：寒暄、身份确认、议程确认
* 需求探索：现状/痛点/影响/决策链（至少命中2–3个）
* 价值呈现：与痛点一一映射，避免“功能堆砌”
* 异议处理：价格/竞品/时机/权限/风险（至少处理1个）
* 下一步推进：明确行动（demo/试点/报价/会议/时间）

**评分例表（你可直接复制到表格）**

* 目标达成（0–5）：是否明确推进到“下一步”并确认时间/责任人
* 需求探索（0–5）：是否问出量化指标/业务影响（时间、成本、风险）
* 价值表达（0–5）：是否用“痛点→能力→收益”结构；是否可量化
* 异议处理（0–5）：是否先共情再澄清；是否给出可选方案
* 节奏掌控（0–5）：是否提问-倾听比例合理；是否打断少
* 合规（0–5）：是否避免夸大承诺/保证效果/敏感对比

**自动提示触发（在线纠偏）建议**

* 连续 2 轮未提问 → 提示补充探索问题
* 出现“保证/一定/百分百” → 合规提醒
* 讨论价格但未回到价值 → 提示“价值锚定”话术

---

### B) 客服陪练 Rubric（投诉/咨询/售后）

**阶段型关键点**

* 情绪安抚：致歉/共情/承诺跟进
* 信息核验：订单/账号/时间/症状/环境
* 解决方案：分支策略（当场解决/升级工单/退款换货）
* 风险与合规：隐私核验、免责边界、口径一致
* 结束确认：复述方案、确认满意度、告知下一次联系时间

**评分点**

* 情绪处理（0–5）：是否先安抚再处理事实；是否避免对抗语气
* 诊断能力（0–5）：是否问到关键信息并排除常见原因
* 方案质量（0–5）：是否给出清晰步骤+预期+时间窗口
* 升级策略（0–5）：何时升级、给用户什么承诺、怎么留痕
* 合规（0–5）：是否做隐私核验；是否避免越权承诺
* 结束收口（0–5）：是否确认问题解决并邀请补充

---

### C) 管理沟通/领导力陪练 Rubric（绩效面谈/辅导/冲突）

**阶段型关键点**

* 建立心理安全：先肯定/表达目的/共识规则
* 对齐事实：用具体行为与例子（SBI/STAR）
* 探索原因：提问而非指责（动机/资源/能力/环境）
* 共创方案：行动计划、资源支持、检查点
* 承诺与跟进：时间表、责任、复盘机制

**评分点**

* 事实表达（0–5）：是否用具体例子而非标签化评价
* 倾听与提问（0–5）：是否反映倾听；是否用开放式问题
* 辅导能力（0–5）：是否帮助对方自己提出方案
* 边界与要求（0–5）：是否明确标准与期望，不含糊
* 冲突处理（0–5）：是否控制情绪、减少防御、面向问题
* 跟进机制（0–5）：是否有检查点与复盘节奏

## 4) 你可以怎么用这套材料做“对标/尽调”

1. **场景状态机怎么做？**（阶段、转移、关键点覆盖、失败回退）
2. **角色五维如何落地？**（情绪状态如何更新、行为决策如何约束LLM）
3. **评估如何做到“可解释”？**（证据片段、扣分归因、稳定性与一致性）
